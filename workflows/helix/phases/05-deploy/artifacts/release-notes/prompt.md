# Release Notes Generation Prompt

Create user-facing release notes that clearly communicate what's new, changed, and improved in this release.

## Storage Location

Store the release notes at: `docs/helix/05-deploy/release-notes.md`

## Purpose

Release notes serve as:
- User communication about changes
- A changelog for version tracking
- A reference for support teams
- A marketing opportunity
- A compliance record

## Key Requirements

### 1. Version & Date

#### Version Information
- Semantic version number
- Release date
- Release type (major/minor/patch)
- Previous version reference

#### Distribution
- Availability (GA, beta, limited)
- Platforms affected
- Regions/languages

### 2. Highlights

#### Key Improvements
- 3-5 most impactful changes
- User benefit focus
- Clear, jargon-free language
- Visual callouts for major features

### 3. New Features

#### Feature Documentation
- Feature name and description
- User benefit statement
- How to access/enable
- Screenshots or GIFs (if applicable)
- Link to detailed documentation

### 4. Bug Fixes

#### Fix Documentation
- Issue description
- User impact
- Resolution summary
- Workaround removal (if any)

### 5. Breaking Changes

#### Change Communication
- What changed
- Why it changed
- User impact assessment
- Required action by users
- Timeline for deprecation

### 6. Migration Guide

#### Upgrade Instructions
- Step-by-step migration
- Configuration changes needed
- Data migration requirements
- Rollback instructions
- Support contact for issues

## Writing Guidelines

### Tone
- User-focused, not developer-focused
- Benefits over features
- Clear and concise
- Professionally friendly

### Structure
- Scannable format
- Bullet points over paragraphs
- Consistent formatting
- Logical grouping

### Accessibility
- Plain language
- No unexplained jargon
- Links to detailed docs
- Contact for questions

## Quality Checklist

Before publishing release notes:
- [ ] Version number correct
- [ ] All changes documented
- [ ] Breaking changes highlighted
- [ ] Migration guide complete
- [ ] Links verified
- [ ] Spelling/grammar checked
- [ ] Screenshots current
- [ ] Support team reviewed

## Integration with Deploy Phase

These notes coordinate with:
1. **GTM Plan** - Messaging alignment
2. **Launch Checklist** - Communication readiness
3. **Deployment Checklist** - Version accuracy

Remember: Release notes are often the first thing users read about an update. Make them clear, helpful, and complete.
